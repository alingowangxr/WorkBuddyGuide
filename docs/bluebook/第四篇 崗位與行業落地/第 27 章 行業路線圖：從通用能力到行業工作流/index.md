# 第 27 章 行業路線圖：從通用能力到行業工作流

行業方案不是把“生成周報、做 PPT”換一個行業名稱。真正的行業化至少包含四層：行業物件、專業流程、系統與資料、責任和監管邊界。

## 識別行業場景的五個問題

1. 工作物件是什麼：訂單、裝置、病例、合同、課程還是內容素材？
2. 流程由哪些崗位和系統共同完成？
3. 哪些欄位、口徑或版本必須保持一致？
4. 錯誤會造成什麼後果，誰擁有最終責任？
5. 哪一步最重複、最耗時，又最容易人工驗收？

只有第五類步驟適合作為首批 AI 試點。

```mermaid
flowchart LR
    A[裝置/生產/質檢記錄] --> B[歸集與欄位校驗]
    B --> C[異常與趨勢分析]
    C --> D[日報/週報/質量報告]
    D --> E[人工確認與閉環]
```

| 場景 | 輸入 | 交付物 | 專業約束 |
|-|-|-|-|
| 生產報表 | ERP 匯出、班組表、停機記錄 | 產量、良率、停機原因 | 班次、物料、工序口徑一致 |
| 質量記錄 | 檢驗表、圖片、標準規範 | 缺陷分類、異常批次、報告草稿 | 不代替質量放行 |
| 裝置知識 | 手冊、維修記錄、故障程式碼 | 排障建議、備件清單 | 安全操作和停機許可權 |
| 供應鏈週報 | 訂單、庫存、交期 | 風險清單、催辦建議 | 資料時點和供應商許可權 |

## 跨境電商

跨境電商的難點不是單一內容生成，而是多平臺、多語言、多幣種、多時區和多規則並存。

| 場景 | WorkBuddy 工作流 | 驗收重點 |
|-|-|-|
| 商品上新 | 產品資料 → 賣點 → 多語種草稿 → 平臺格式 | 引數、停用詞、商標和本地表達 |
| 客服與評價 | 工單聚類 → 回覆草稿 → FAQ 更新 | 訂單事實、退款承諾、語氣 |
| 經營覆盤 | 訂單/廣告/庫存 → 統一口徑 → 利潤與異常 | 匯率、時區、歸因視窗 |
| 供應鏈協同 | 採購、交期、物流 → 風險清單 | 單號、時間、責任人 |

## 金融、諮詢與財稅

這類行業的核心資產是證據、分析口徑和專業判斷。AI 適合縮短資料處理時間，不適合隱藏來源後直接給結論。

| 場景 | 基礎版 | 進階版 | 人工關口 |
|-|-|-|-|
| 行業研究 | 來源表和資料摘要 | 證據矩陣、定期跟蹤 | 觀點與投資含義 |
| 諮詢專案 | 訪談紀要、材料盤點 | 假設樹、分析底稿、報告 | 建議與客戶承諾 |
| 財稅材料 | 單據識別、欄位核對 | 申報資料清單、差異檢查 | 申報和稅務判斷 |
| 經營分析 | 多表彙總 | 指標解釋、異常歸因 | 會計口徑和管理決策 |

## 醫療醫藥

醫療醫藥強調證據等級、版本控制、隱私和專業資質。首批場景應避開診斷和治療決定。

| 場景 | 可由 AI 輔助 | 不應自動完成 |
|-|-|-|
| 文獻整理 | 檢索結果去重、摘要、證據表 | 臨床結論和治療建議 |
| 生產質量 | 批記錄完整性檢查、培訓材料 | 批次放行和偏差定責 |
| 醫學/市場材料 | 來源核對、停用表述提示 | 宣傳合規最終批准 |
| 內部知識 | 制度問答、版本定位 | 超許可權訪問患者或試驗資料 |



## 文化傳媒與廣告

影片團隊案例已經展示了多 Agent 內容流水線。行業落地時，還需要在創作效率之外加入版權、品牌和釋出控制。

```mermaid
flowchart LR
    A[熱點與客戶 brief] --> B[選題評分]
    B --> C[指令碼與分鏡]
    C --> D[素材與製作]
    D --> E[事實/版權/品牌稽核]
    E --> F[人工釋出]
    F --> G[資料覆盤]
```

可沉澱資產，品牌語氣、停用詞、事實來源庫、授權素材庫、平臺格式模板、選題評分表、指令碼 Skill、分鏡模板和覆盤指標。

## 建築裝飾與科技服務

這類行業以專案為單位，材料多、版本多、責任鏈長。WorkBuddy 適合做資料盤點、標書初稿、合同差異和專案週報。

| 場景 | 交付物 | 風險控制 |
|-|-|-|
| 招投標 | 響應矩陣、材料清單、標書初稿 | 逐條對應招標檔案，不虛構資質 |
| 方案設計 | 需求摘要、方案結構、彙報材料 | 專業計算和圖紙需資質人員確認 |
| 專案管理 | 紀要、進度、風險和變更清單 | 版本、簽字和責任主體可追溯 |
| 合同與報價 | 條款差異、報價表檢查 | 金額、稅率和承諾人工複核 |
