# 用 WorkBuddy 公眾號 Skill 一鍵排版併發布到微信公眾號草稿箱

## 場景描述

寫公眾號文章，最耗時的往往不是內容本身，而是排版。在微信後臺編輯器裡調整字號、行距、縮排、圖片位置，經常要反覆預覽和修改。整篇文章從寫完後到真正能釋出，排版時間可能和寫作時間一樣長。用 WorkBuddy 把這件事變成兩步：用 Markdown 寫好內容，剩下的排版和釋出交給 Skill 自動完成。

我自己平時寫公眾號最大的問題就是排版，真的對排版深惡痛絕，尤其是圖片外鏈的處理。之前我的公眾號排版流程是這樣的：markdown寫好內容->第三方編輯器美化->貼上到微信公眾號草稿中（最大的問題就在這，我的文章中會放很多圖片，但是從編輯器中貼上過來的圖片經常掛掉，因為公眾號不支援第三方圖床）->逐一貼上圖片->預覽、修改->釋出。

而現在有了Wechat Publisher可以一鍵排版，自動上傳圖片到公眾號草稿箱，真的是節省了我半個多小時的時間。

## 想要完成的任務

完成後，WorkBuddy 應當能夠：

1. 讀取一篇 Markdown 格式的文章（標題、正文、圖片、程式碼塊）。
2. 自動將 Markdown 轉換為微信公眾號相容的排版格式，包括程式碼高亮、主題樣式和 Mac 風格程式碼塊。
3. 自動將文章中的本地圖片和網路圖片上傳至微信圖床。
4. 一鍵將排版後的文章推送到公眾號草稿箱。
5. 在公眾號後臺預覽或微調後即可群發。

## 使用的 Skill

| Skill            | 用途                                              | 來源或安裝方式                   |
| ---------------- | ------------------------------------------------- | -------------------------------- |
| wechat-publisher | Markdown 轉公眾號排版，一鍵上傳圖片並推送到草稿箱 | WorkBuddy 內建推薦市場，直接安裝 |

wechat-publisher 基於開源專案 [wenyan-cli](https://github.com/caol64/wenyan-cli)，內建多套排版主題和程式碼高亮方案，支援自動圖片上傳和草稿箱推送。

## 前置條件

- WorkBuddy 可以正常使用。
- 已經擁有微信公眾號（訂閱號或服務號），並具備開發者許可權。
- **已獲取微信公眾號的 AppID 和 AppSecret（在公眾號後臺「設定 → 開發 → 基本配置」中檢視）**。
- 本機公網 IP 已新增到公眾號後臺的 IP 白名單中（最好用WiFi而非手機熱點，因為熱點IP每次都會變，那麼每次就要重複新增白名單IP）。
- 有一篇用 Markdown 寫好的文章（含標題和封面圖）。

## 在 WorkBuddy 中的操作

### 第一步：安裝 wechat-publisher Skill

在 WorkBuddy 中新建任務，輸入：

```text
幫我安裝公眾號排版 Skill：wechat-publisher
**或者直接去技能搜尋框搜尋：wechat-publisher** 這個簡單直接
```
![skill安裝](assets/skill-install.png)
WorkBuddy 會從推薦市場中搜索並安裝這個 Skill。安裝完成後，可以確認一下 Skill 是否就位：

```text
wechat-publisher Skill 已經安裝好了嗎？告訴我它能做什麼。
```


### 第二步：配置公眾號憑證

wechat-publisher 需要公眾號的 AppID 和 AppSecret 才能呼叫微信 API。推薦使用 wenyan 自帶的憑證管理工具進行配置，憑證會持久化儲存到本地配置檔案：

```text
幫我執行 wenyan credential -s，我需要配置公眾號的 AppID 和 AppSecret。
```

WorkBuddy 會引導你逐步輸入 AppID 和 AppSecret，配置完成後對話方塊確認檢視憑證檔案是否已經儲存成功。


> **注意**：在配置憑證之前，務必先登入公眾號後臺（設定 → 開發 → 基本配置 → IP 白名單），將本機的公網 IP 新增到白名單中。可以用 `curl ifconfig.me` 檢視當前公網 IP。

### 第三步：準備 Markdown 文章

文章的 Markdown 檔案需要包含 frontmatter，至少要有 `title`（標題）和 `cover`（封面圖路徑）。封面圖可以是本地檔案或網路 URL。

```markdown
---
title: 我的技術文章
cover: ./assets/cover.jpg
---

# 正文開始

這是文章內容...
```

可以直接在 WorkBuddy 中讓 AI 幫你寫好文章，也可以把自己寫好的 Markdown 檔案拖進對話。如果還沒有封面圖，還可以讓 WorkBuddy 幫你生成一張：

```text
幫我生成一張適用於公眾號技術文章封面的圖片，主題是關於 AI 輔助寫作的。
```


### 第四步：執行排版與釋出

文章準備好後，告訴 WorkBuddy 執行釋出：

```text
使用 wechat-publisher Skill，把這篇文章釋出到公眾號草稿箱。

要求：
- 使用 lapis 主題
- 程式碼高亮用 solarized-light
- 啟用 Mac 風格程式碼塊
- AppID 用我之前配置的那個
```

WorkBuddy 會自動完成以下步驟：

1. 檢查 Markdown 檔案的 frontmatter 是否完整
2. 將文章中的圖片上傳到微信圖床
3. 應用所選主題和程式碼高亮樣式進行排版轉換
4. 呼叫微信 API 將排版後的文章推送到草稿箱


### 第五步：在公眾號後臺預覽與群發

釋出到草稿箱後，登入微信公眾號後臺，在「內容管理 → 草稿箱」中就能看到排版好的文章。在後臺預覽確認無誤後，即可群發。


## 提示詞或任務指令

下面是經過脫敏、可以直接複用的任務指令：

```text
使用 wechat-publisher Skill，把我寫好的 Markdown 文章釋出到公眾號草稿箱。

具體要求：
1. 文章檔案路徑：D:/articles/my-post.md
2. 使用 lapis 主題和 solarized-light 程式碼高亮
3. 啟用 Mac 風格程式碼塊裝飾
4. 文章中的圖片（本地路徑和網路 URL）全部自動上傳到微信圖床
5. 釋出前確認 frontmatter 包含 title 和 cover
6. AppID 使用我之前通過 wenyan credential -s 配置的那個

釋出成功後，告訴我如何到公眾號後臺預覽這篇文章。
```

## 在 WorkBuddy 中的效果

這一套流程走完後，你得到的是：

- 一篇已經排版完成的公眾號草稿，不需要在微信後臺編輯器中手動調整字號、間距、圖片位置。
- 文章中的程式碼塊帶有語法高亮和 Mac 風格裝飾，比微信原生編輯器效果好得多。
- 所有圖片已自動上傳到微信圖床，不會出現圖片丟失或外鏈失效的問題。
- 在公眾號後臺稍作預覽即可直接群發。

## 常見問題

### 釋出時報 "IP 不在白名單"

這是最常見的問題。登入公眾號後臺，在「設定 → 開發 → 基本配置 → IP 白名單」中添加當前機器的公網 IP。用 `curl ifconfig.me` 可以檢視公網 IP。

### 報錯 "未能找到文章封面"

檢查 Markdown 檔案的 frontmatter 中是否包含了 `cover` 欄位。wenyan-cli 強制要求 title 和 cover 兩個欄位缺一不可。封面圖路徑建議使用絕對路徑（Windows 下如 `D:/photos/cover.jpg`）。

### 圖片上傳失敗

確保圖片路徑正確。如果 Markdown 中的圖片使用相對路徑，需要在 Markdown 檔案所在目錄執行釋出命令。或者改用絕對路徑引用圖片。

### 想換一種排版風格

wechat-publisher 支援多種內建主題。可以在釋出命令中更換 `-t` 引數，例如：

- `-t lapis`：青金石主題（推薦）
- `-t phycat`：物理貓主題
- `-t default`：預設主題

程式碼高亮也可以通過 `-h` 引數切換，如 `-h github`、`-h monokai`、`-h dracula` 等。

**當然，這些指令記記不住都沒有關係，直接用自然語言描述，告訴AI你想做的事情，它可以幫你切換主題。**

## 安全與限制

- AppID 和 AppSecret 是公眾號的核心憑證，不要分享給他人，也不要在公開場合暴露。
- 推薦使用 `wenyan credential -s` 配置憑證（憑證儲存在本地加密檔案中），避免將金鑰寫在命令或環境變數裡。
- 公眾號 IP 白名單建議只新增必要的 IP，並定期檢查。
- 文章推送到的是草稿箱而非直接群發，釋出前仍需人工稽核內容。
- 如果文章包含公司內部或敏感資訊，釋出前先做脫敏處理。

## 可以怎樣複用

這個 Case 的核心能力是「Markdown 寫作 + 自動排版釋出」。替換文章內容，就能適配不同的公眾號創作場景：

- **技術部落格**：Markdown 寫技術教程，程式碼塊自動高亮，推送到草稿箱稽核後群發。
- **團隊週報**：用模板和 Markdown 整理週報內容，自動排版後分享到團隊公眾號。
- **多號管理**：通過 `--app-id` 引數切換不同公眾號，一個 Skill 管理多個號的草稿釋出。
- **定期專欄**：結合 WorkBuddy 自動化任務，把固定格式的文章模板設為每日或每週重複執行，實現定期更新。
