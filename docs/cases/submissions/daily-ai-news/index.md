# 【示例】用 WorkBuddy 自動整理每日 AI 資訊

## 場景描述

AI 資訊更新很快，同一事件經常出現在多個媒體和資訊源中。逐個網站瀏覽不僅耗時，也容易錯過模型釋出、產品更新、開源專案和行業變化。

這個案例適合需要持續關注 AI 行業的產品、研發、運營、投資和內容團隊。我們希望把“到處找新聞”變成一次可以重複執行的 WorkBuddy 任務。

## 想要完成的任務

完成後，WorkBuddy 應當能夠：

1. 查詢指定時間範圍內的 AI 行業動態。
2. 按公司、模型、產品或主題篩選內容。
3. 為每條資訊保留標題、摘要、釋出時間和來源連結。
4. 把大量候選資訊整理成更容易閱讀的熱點報告。
5. 根據任務要求重點關注大模型、產品、開源專案或論文等方向。

## AIHot Skill 能做什麼

AIHot 是面向 AI 動態的資訊類 Skill，適合查詢和整理模型、產品、行業、論文與開源專案等方向的近期內容。它可以作為 WorkBuddy 的現成資訊源，根據你的任務描述返回相關資訊和原始連結。

| 用法 | 任務示例 | 適合誰 |
| --- | --- | --- |
| 公司動態追蹤 | 最近 7 天 OpenAI 釋出了什麼 | 產品、研發、投資 |
| 每日熱點整理 | 總結今日 AI 大模型熱點 | AI 團隊、內容團隊 |
| 專題檢索 | 查詢最近一週多模態模型相關動態 | 研究、產品規劃 |
| 來源補充 | 給每條結論保留原始報道連結 | 需要進一步核查的讀者 |

AIHot 負責提供和檢索候選資訊；哪些內容值得重點關注、怎樣分類和以什麼格式輸出，由你寫給 WorkBuddy 的任務描述決定。

## 安裝前準備

- 已經可以正常使用 WorkBuddy。
- WorkBuddy 可以訪問外部網頁。
- AIHot Skill 地址：`https://aihot.virxact.com/aihot-skill/`

AIHot 來自外部地址。安裝前應檢視來源域名和 Skill 內容，並讓 WorkBuddy 先執行安全檢查。不要安裝來源不明、要求異常許可權或包含可疑指令碼的 Skill。

## 在 WorkBuddy 中安裝 AIHot Skill

開啟一個新的 WorkBuddy 任務，直接輸入：

```text
幫我安裝這個 skill：https://aihot.virxact.com/aihot-skill/
```

WorkBuddy 會先獲取 Skill 內容，並呼叫 Skill 安全檢查流程。確認來源、許可權和將要執行的內容沒有異常後，再繼續安裝。

![WorkBuddy 獲取 AIHot Skill 並執行安全檢查](./assets/aihot-install.png)

安裝完成後，可以繼續問：

```text
請告訴我 AIHot Skill 已經安裝到哪裡、它能完成哪些任務，以及呼叫時是否需要額外賬號或許可權。
```

這一步可以幫助你確認 Skill 是否真的可用，而不是隻看到安裝任務結束。

## 任務一：查詢最近 OpenAI 動態

最簡單的任務描述是：

```text
請看一下最近 OpenAI 釋出了什麼新東西。
```

為了讓結果更穩定、方便核查，推薦寫得更明確：

```text
使用 AIHot Skill 查詢最近 7 天與 OpenAI 相關的動態。

要求：
1. 優先整理 OpenAI 官方釋出、模型與產品更新、API 變化和重要行業動態；
2. 合併明顯重複的同一事件；
3. 每條內容包含標題、釋出時間、兩句話摘要和原始來源連結；
4. 按“官方動態、產品與模型、行業影響”分類；
5. 不確定的資訊請明確標記，不要補寫沒有來源支援的結論。
```

實際執行後，WorkBuddy 返回了最近 7 天的 OpenAI 相關內容，併為每條資訊保留標題、時間、摘要和 AIHot 原始連結：

![AIHot 查詢最近 7 天 OpenAI 動態的結果](./assets/aihot-openai-result.png)

## 任務二：生成今日 AI 大模型熱點總結

如果希望得到一份更接近日報的結果，可以輸入：

```text
使用 AIHot Skill 總結今天的 AI 熱點新聞，重點關注 AI 大模型方向。

請按下面結構輸出：
1. 今日最熱：列出最值得關注的 3 條動態；
2. 大模型新發布：整理最近 7 天值得關注的模型、產品和開源專案；
3. 每條包含一句話摘要、為什麼值得關注和來源連結；
4. 合併重複報道，並註明有多少個資訊源在報道同一事件；
5. 最後用一句話總結今天的大模型行業主線。
```

實際結果按照熱點和大模型方向進行了整理，保留了具體來源連結，並給出一句話結論：

![AIHot 生成今日 AI 大模型熱點總結](./assets/aihot-daily-report.png)

## 怎樣把任務描述寫得更好

一個穩定的 AIHot 任務描述，至少應該明確四件事：

| 要素 | 寫法示例 |
| --- | --- |
| 時間範圍 | 今天、最近 24 小時、最近 7 天 |
| 關注主題 | OpenAI、大模型、多模態、AI Agent、開源專案 |
| 篩選標準 | 只保留重要更新、合併重複事件、優先官方來源 |
| 輸出格式 | 標題、摘要、時間、影響判斷、來源連結 |

如果只說“總結 AI 新聞”，WorkBuddy 仍然可以執行，但結果的數量、範圍和格式不夠穩定。任務描述越明確，輸出越容易複用和驗收。

## 最終效果

通過 AIHot Skill，WorkBuddy 把分散的 AI 資訊整理成了兩類可以直接閱讀的結果：

- 圍繞特定公司或主題的近期動態列表。
- 經過篩選、分類和去重的 AI 熱點總結。

每條資訊都保留來源連結，讀者可以繼續開啟原文核查，而不是隻依賴模型生成的摘要。

## 驗收標準

- AIHot Skill 已成功安裝，並能在新任務中被呼叫。
- 查詢結果符合任務指定的時間範圍和主題。
- 每條重要資訊都保留了可以繼續訪問的來源連結。
- 相同事件不會被當作多條重點新聞重複呈現。
- 摘要沒有加入來源無法支援的數字和結論。
- 熱點報告的分類與任務描述一致。

## 常見問題

### 安裝後沒有呼叫 AIHot

在任務中明確寫出“使用 AIHot Skill”，並讓 WorkBuddy 說明它實際呼叫了哪個 Skill。如果仍然不可用，檢查 Skill 是否安裝成功以及當前工作區是否能看到它。

### 返回的資訊太多

縮短時間範圍，限定公司或主題，並明確只保留 5～8 條最重要內容。

### 結果看起來像普通摘要

要求保留髮布時間、來源名稱和原始連結，並明確寫出分類、去重和影響判斷規則。

## 安全與限制

- AIHot 是資訊來源，不代表所有內容都已經完成事實核查。
- 對重要結論，應繼續開啟原始連結，優先核對官方來源。
- 新聞內容會隨時間變化，相同任務在不同日期得到的結果也會不同。
- 安裝外部 Skill 前應檢查來源、許可權和指令碼內容。
- 不要把未經核實的熱點摘要直接用於正式對外發布。

## 可以怎樣複用

把任務中的主題替換掉，就可以快速得到不同方向的資訊報告，例如：

```text
使用 AIHot Skill 查詢最近 7 天 AI Agent 相關動態，重點關注新產品、開源框架和企業落地案例。每條保留來源連結，最後總結 3 個值得團隊繼續研究的方向。
```

也可以在確認單次任務穩定後，再結合 WorkBuddy 自動化任務，把相同提示詞設定為每日或每週重複執行。自動化屬於後續擴充套件，這個 Case 首先確保 AIHot 的安裝、呼叫和輸出可以獨立跑通。
